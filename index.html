Form Example

Forms can be very difficult to understand if the mark-up isn't semantic and the labels don't give enough context. Make sure that labels are set and that fieldsets and legends are being used to group related form items, this is particularly important with checkbox and radio inputs. With error and success states, try to avoid relying on just colour to convey meaning - in the form below the colour is paired with iconography and text.

Note: I have used FontAwesome because I didn't have any assets, icon fonts can be an accessibility no-no because of how they render with screen readers: Further information about icon fonts

Accordion

Accordions can be tricky for some users who may use screen readers or assisstive tech to navigate websites. By design, accordion content is hidden and requires users to click to reveal the content. If an accordion isn't built with accessibility in mind, it can be difficult for users to know it's an accordion or they may know it's an accordion but be unable to interact with it.

Modal

Modals can be problematic for some users. Can be confusing, sometimes the content can't be accessed by the screenreader The below example saves the users last focus from before the modal is opened and automatically focuses inside the modal. When the modal is closed, the focus is returned to the last focused point, this prevents users from losing their place on the page. The below modal also restricts users from tabbing outside of the modal whilst it is active, this is to prevent keyboard only users accidentally from tabbing out of the modal without realising.

Breadcrumb

Breadcrumb menus could be confusing for people who rely on screen readers so it is important to add an aria-label="breadcrumb" to the nav, to let the screen reader know what it is and announce it to the user. It is also important to add aria-current="page" to the last element, so that when it is read out, 'current page' is appended to it. eg, "About, current page". I have also used an ordered list as these links are meant to be represented in a hierarchical structure

  1. Index
  2. Sub Page
  3. Current Sub Page

Pagination

Pagination can have similar issues to breadcrumbs, so it is important to use aria tags to help screen readers interpret the content. With the aria-label on each link, the SR will say 'link, go to page 2' rather than 'link, 2' which may be difficult to understand with no context.
